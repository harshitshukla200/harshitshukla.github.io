Engineering Decision Log

Real-world architectural trade-offs and choices

Architecture

Why Apache Kafka?

Context: Needed to handle high-throughput real-time events for TemporalAI.

Decision: Chose Kafka over RabbitMQ/Redis PubSub.

Why: Kafka provides superior durability, event replay capabilities, and high throughput needed for real-time AI anomaly streams. The trade-off was increased operational complexity, mitigated via Docker Compose.

Backend

FastAPI vs Django

Context: Building APIs for AI-heavy microservices.

Decision: Migrated from Django to FastAPI for core AI services.

Why: FastAPI's native async support perfectly complements IO-bound LLM API calls and Kafka interactions. It reduced API response times by 20% compared to synchronous Django views.

Database

NoSQL (Firebase) vs SQL

Context: Storing dynamic chat histories and real-time anomaly alerts.

Decision: Used Firebase Firestore.

Why: Provided out-of-the-box real-time listeners for the frontend to update UI instantly without complex WebSockets, while handling unstructured JSON logs effectively.
